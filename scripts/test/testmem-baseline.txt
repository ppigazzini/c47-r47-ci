# c43 per-test pool/GMP growth baseline.
# Cases on upstream master where a test grew the RAM pool past the high-water.
# run-testmem.sh fails on any growth case NOT listed here.
# Format: <file>|<IN parameter line>
#
# Regenerated against upstream master 1624e09b3 (gcc, full corpus). Every case
# below grows only the RAM pool (gmp=+0, no GMP/bignum leak) and the residue is a
# one-shot allocation retained for the rest of the run, not a per-iteration leak:
#
#   eq_cov/deriv_cov/solve_cov/integrate_cov/pgm_solve_cov  the formula-coverage
#                     drivers leave shared state the per-test In: reset does not
#                     clear (a formula in the pool, named variables A and X, the
#                     reserved TVM registers, an appended program).
#   program_flow_cov/shortint_restore_cov  the flow-control and short-integer
#                     restore drivers append/clear programs and save/reload state,
#                     leaving one-shot residue like the other coverage drivers.
#   graphs_cov.txt    the plot-regression cases load the G1..G4 graph programs and
#                     render bitmaps, state kept for the rest of the run.
#
# The normalize in run-testmem.sh keys on <file>|<case>, not on the growth
# magnitude, so only a brand-new growing case fails the gate.
#
# Regenerating against 1624e09b3 dropped 9 cases that no longer grow (4 eq_cov,
# 2 integrate_cov, 2 program_flow_cov, 1 shortint_restore_cov: 20 -> 11). The
# corpus input files are byte-identical to the a361b6797 pin, so the growth
# stopped in the engine within a361b6797..1624e09b3, not in the tests. That range
# includes f94eba918 "each test file can run separately without errors" and the
# solver/integrator changes 3bd0f13ea and c4a48ab49; which one retired which case
# is not bisected here. These are removals of findings that no longer reproduce,
# not a re-blessing: the gate still fails on any new growing case (verified with
# an injected control).
#
# 6 cases added against a39865f53 (11 -> 17), all pool-only (gmp=+0), from two
# upstream test additions after the 1624e09b3 pin:
#
#   nested_cov.txt    new file (56212cbf6, the AN0022 nested engine corpus).
#                     FARG=0 is fnLoadNestedPgmsCov appending the AN0022
#                     programs (+1504, retained like the graphs_cov loaders);
#                     the four engine cases are +24..+36 of solver/integrator
#                     residue in the pool, the class the solve_cov and
#                     integrate_cov entries already bless.
#   readp_cov.txt     the program-file screening added in 09280ff24 + 55a835764
#                     (over-long-label refusal, screen-before-load) grew the
#                     driver's one-shot residue to +108.
#
# Measured with a doubled test list (each file run twice in one process): the
# second pass adds only the by-design program appends and +12/-0 engine residue,
# no repeated growth of the same magnitude - one-shot retention, not a
# per-iteration leak.

eq_cov.txt|IN: FARG=1 RX=REAL:"0"
eq_cov.txt|IN: FARG=2 RX=REAL:"0"

graphs_cov.txt|IN: FARG=1

graphs_cov.txt|IN: FL_ASLIFT=1 RZ=REAL:"-5" RY=REAL:"5" RX=STRI:"SIN(X)" PGM="G1"
graphs_cov.txt|IN: FL_ASLIFT=1 RZ=REAL:"-5" RY=REAL:"5" RX=STRI:"X^2" PGM="G1"
