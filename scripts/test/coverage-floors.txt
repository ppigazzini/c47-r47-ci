# Per-sector coverage ratchet floors (line %) for the c47 coverage lane.
# run-coverage.sh gates on these when SECTOR_GATE=1. The overall global floor is
# the true whole-tree line percentage, gated separately by COVERAGE_MIN (this
# file's sectors sum only the files a sector matches, a different denominator).
#
# Each floor sits a point or two below the measured baseline so gcov noise does
# not flap the gate, but a real regression fails CI. Raise a floor when the
# corpus lifts a sector; lowering one is an explicit, documented acceptance of a
# regression.
#
# Baseline: upstream master ef694abb0, gcc 13, full corpus + keyscan + leakscan,
# sample programs staged. Sectors measured -> floor:
#
#   core math          85.02% -> 84      matrix             69.70% -> 68
#   programming / cli  39.65% -> 38      serializers        68.94% -> 67
#   strings / alpha    27.84% -> 26
#
# Since 9c9ac6549: programming/cli rose 37.06 -> 39.65 and core math 84.88 ->
# 85.02 as the new coverage corpus (bits/flags/statsx/regmgmt/histo/clearvars
# _cov) exercises more of the bit-op, flag, register and stats paths. Their
# floors ratchet 35 -> 38 and 83 -> 84; matrix, serializers and strings/alpha
# are unchanged.
#
# Gated: the CLI-critical sectors (serializers, program management, parser/string
# paths, core math) plus matrix. Left ungated by design (host-test ceilings /
# lower priority): solver/graph, input/editors, ui/display/printing, addons/dmcp.
core math = 84
matrix = 68
programming / cli = 38
serializers = 67
strings / alpha = 26
